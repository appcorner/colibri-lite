# M4.2 Optimization Invariants

## Purpose

This checklist is the correctness gate for every optimization after M4.2. It
does not authorize an optimization. The active task and milestone plan must do
that separately.

An optimization must preserve these invariants or stop for review before its
implementation is retained. New backends or dtypes that cannot preserve an
exact F32 contract require a separately versioned comparison contract and must
not weaken the M4.2 registry.

## Frozen Identity and Layout

- [ ] Model remains `Qwen/Qwen3-30B-A3B` revision
  `ad44e777bcd18fa416d9da3bd8f70d33ebb85d39`.
- [ ] Canonical root manifest SHA-256 remains
  `f133d733612840ad691d637732d4ef2de1e0242c4bb1d92521b49dfcfb1b8cd2`.
- [ ] Canonical model artifacts and pinned source remain read-only.
- [ ] BF16-to-F32 conversion remains exact at the bit level.
- [ ] Tensor names, dtypes, shapes, offsets, byte lengths, orientations, and
  shard identities remain exact.
- [ ] Packed expert payload order remains gate, up, down with the frozen matrix
  orientations and offsets.
- [ ] Only selected experts are loaded for sparse execution.

## Numerical and Semantic Invariants

- [ ] Ordered scalar Python F32 and Rust RMSNorm remain bit-exact, including
  reduction, epsilon placement, and `(x * reciprocal_rms) * weight`.
- [ ] The provisional PyTorch same-input RMSNorm guard remains both 8 ULP and
  `5e-7`; it is not enlarged automatically for a new implementation.
- [ ] Router ranking remains higher score first and lower expert ID first for
  exact ties.
- [ ] Transformers F32 remains the primary Rust F32 numerical oracle; BF16 is
  classified independently.
- [ ] Frozen F32 router IDs match at all recorded guard layers and remain
  `exact_match_safe` under the documented margin rule.
- [ ] Selected intermediate identities and membership remain exact.
- [ ] Test-trace down projections remain bit-identical to the normal Rust
  expert path.
- [ ] All layer-specific, selected-case, and generation budgets in the M4.2
  registry pass without being widened merely to accept a change.
- [ ] Vocabulary top-20 ordering and generated IDs `[1096, 374]` remain frozen
  for prompt `[9707, 11, 1879, 0]`.
- [ ] Every generated F32 selection remains `exact_match_safe`.
- [ ] NaN and infinity counts remain zero at all guarded outputs and logits.

## KV Cache and Memory Invariants

- [ ] The frozen regression retains 48 KV layers, capacity/final length 6,
  per-layer K/V shape `[6, 4, 128]`, and 1,179,648 payload bytes.
- [ ] Token positions advance exactly once and previous positions are never
  overwritten.
- [ ] Primary cached decode never silently falls back to full recomputation.
- [ ] Cached, incremental-reference, and full-recompute argmax results agree.
- [ ] Cache allocations remain fixed for the configured capacity unless a
  reviewed cache contract explicitly replaces that guarantee.
- [ ] Expert-cache resident bytes never exceed the configured byte budget.
- [ ] Cache requests, hits, misses, loads, evictions, and bytes-read counters
  reconcile.
- [ ] Explicit memory remains bounded and its model is distinguished from
  process working set/private bytes.

## Evidence Invariants

- [ ] `m4.2-tolerance-contract-registry-v1.json` passes its consistency and
  evidence-hash tests.
- [ ] Existing evidence schemas remain readable and deterministic fields remain
  stable.
- [ ] An intentional schema change is versioned and documented; evidence is not
  overwritten in place.
- [ ] The first differing checkpoint is reported before later checkpoints are
  accepted.
- [ ] Successful temporary runs are removed under the flat-run policy.

## Mandatory Regression Suite

Every optimization runs:

```powershell
cargo fmt --all --check
cargo check --workspace
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings
cargo run -p clr-cli
python -m compileall -q python scripts
python -m unittest python.reference.test_m4_2_tolerance_registry
```

It also runs warning-free feature Clippy and the frozen optimized M4.2-04
generation regression with byte-identical deterministic evidence.

### Larger expert cache or expert prefetch

- Run all `clr-storage` cache-budget, pinning, eviction, release, and repeated
  access tests.
- Run streaming resident-versus-streaming and repeated cached-decode tests.
- Run router, selected-intermediate, and M4.2-04 generation regressions.
- Reconcile changed hit/load/eviction counters without changing selected
  experts or numerical evidence.
- Re-measure bounded process and explicit memory before retaining the change.

### Memory mapping, buffered I/O, or direct I/O

- Run artifact range, truncation, corruption, file-lifetime, and handle-release
  tests on Windows.
- Revalidate canonical hashes, exact selected tensor values, offsets, and
  packed expert payloads.
- Run the complete selected-intermediate and generation regressions.
- Preserve logical-read semantics or introduce a separately named physical-I/O
  metric; never conflate them.

### Quantization

- Preserve the canonical F32 artifact and registry as the comparison baseline.
- Define a new versioned format, oracle path, per-checkpoint degradation
  budgets, and stop conditions before implementation evidence is accepted.
- Run router margins, selected intermediates, full logits, token selection,
  cache, memory, and artifact provenance checks.
- Never rewrite an M4.2 F32 tolerance to describe quantized error.

### SIMD or alternative accumulation algorithms

- Run ordered internal-kernel tests and same-input diagnostics first.
- Run all propagated router checkpoints, selected expert intermediates, final
  RMSNorm, full logits, and generation regressions.
- If operation ordering changes exact internal results, stop for a new reviewed
  backend contract instead of weakening the scalar contract.

### Parallel expert execution

- Prove deterministic expert occurrence grouping and aggregation order.
- Validate every selected intermediate, routing-weight association, aggregate,
  residual, and final block checkpoint.
- Repeat evidence generation to prove scheduling does not change bytes.
- Enforce a modeled and measured peak-memory budget for simultaneous experts.

## Review Gate

An optimization is rejected or paused when any invariant fails, an evidence
hash changes unexpectedly, a budget must be widened without diagnosis, or a
new unsafe/dependency/artifact/public contract is required. Performance gain
alone is not evidence of correctness.
