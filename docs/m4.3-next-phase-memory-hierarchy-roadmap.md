# Next-Phase Memory-Hierarchy Roadmap

## Experiment boundary

The first experiment is simulation only. It uses the recorded short-generation
and intermediate expert request traces and does not modify Rust runtime code,
ExpertStore, cache capacity, artifacts, or numerical arithmetic.

## Matrix

Evaluate RAM budgets of 1, 2, 4, 8, 16, 24, and 32 binary GiB for each
configuration:

1. resident dense weights plus bounded expert cache;
2. streamed dense weights plus bounded expert cache;
3. resident dense weights with expert-cache-only residency.

For each trace and budget, simulate the exact request order and report cache
hits, misses, loads, evictions, peak resident bytes, unique experts, bytes
avoided, logical read amplification, and estimated explicit RAM. Counters must
reconcile with capacity and insertions. Dense residency must be accounted for
separately from expert residency.

## Review gates

The simulation passes review only when it preserves frozen F32 correctness
references, predicts a material reduction in logical reads per token, produces
measurable hits at a bounded budget, avoids artifact duplication, and reports
all seven budget points. Throughput is not claimed from simulation.

Only after review may a minimal memory-path prototype be considered. That
prototype must rerun the F32 Tier C/B/A regression and compare memory, I/O,
latency, and dependency deltas against M4.2-05. No mmap, prefetch, SIMD,
threading, quantization, or accelerator work is implied by this roadmap.
