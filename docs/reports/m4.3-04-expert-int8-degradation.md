# M4.3-04 Expert INT8 Degradation Study

## Decision

The selected symmetric INT8 per-output-channel expert format is classified as
`quality_risk`. It is not accepted for a runtime prototype. The local Tier C
expert gates pass, but propagated full-forward degradation is too large and the
short fixtures include a Tier-B argmax change under a numerically ambiguous
margin. Tier A still produces `[1096, 374]`, but generated IDs alone are not an
acceptance gate.

No production runtime, ExpertStore, cache, artifact, router, dense tensor,
activation, or accumulation path changed. The simulation reconstructs each
selected expert projection from deterministic INT8 values and F32 row scales,
then calls the existing F32 matrix operation.

## Configuration and Coverage

- Expert gate/up/down: symmetric INT8 per output channel, F32 scales,
  nearest-even rounding, `[-127, 127]` saturation
- Expert activations and accumulation: F32
- Embedding, attention Q/K/V/O, router, all norms, routing weights, residuals,
  softmax, final norm, LM head, and KV cache: F32
- Tier C: 8 representative M4.2-03 expert cases
- Tier B: all six frozen fixtures
- Tier A: prompt `[9707, 11, 1879, 0]`, two generated tokens
- Guard layers: 0, 1, 8, 16, 24, 32, 40, 47

The canonical F32 artifact and pinned source remained read-only. No complete
quantized artifact was created.

## Failure Localization

The earliest local operation remains within the M4.3-02 Tier-C gates. The first
propagated risk is Layer 1's combined MoE/final block error; later large Layer
47 error is accumulated incoming drift amplified by expert and router paths.
No tolerance was widened and the F32 registry was not changed. A future
investigation should first test lower-error group-wise quantization or a
layer-range diagnostic; it must not change router or RMSNorm precision to hide
the drift.

## Classification and Future Gates

The candidate is `quality_risk`, not `accepted_for_runtime_prototype` and not
`rejected` as a format forever. Future prototype work requires a lower-error
variant or a separately reviewed contract, while preserving F32 router,
normalization, residual, routing-weight, and accumulation behavior.

The F32 tolerance registry and baseline invariants remain unchanged. The
provisional degradation record is
`models/qwen3-30b-a3b/m4.3-04-provisional-degradation-gates-v1.json`.
