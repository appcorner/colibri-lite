## Policy and tooling

`docs/temp-artifact-policy.md` now requires one flat unique run directory,
read-only canonical source/artifact reuse, atomic final promotion, automatic
successful-run removal, explicit bounded debug retention, no nested temp roots,
preflight/post-task disk accounting, and rename/hardlink/junction/root-override
relocation tests instead of full copies.

`scripts/cleanup_temp_artifacts.py` consumes the reviewed JSON plan and defaults
to dry-run. Apply mode requires `--apply`. It refuses protected/canonical path
overlap, missing canonical references, nested candidates, symlink/junction
traversal, non-deletable classifications, and any count/byte/link drift.

Five safety tests prove dry-run immutability, explicit-apply behavior in an
isolated fixture, canonical root/reference protection, source/parent
protection, and drift refusal.
