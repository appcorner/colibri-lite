# M4.1-05 Dense Conversion Report

## Scope

Converted every non-expert tensor from pinned `Qwen/Qwen3-30B-A3B` revision
`ad44e777bcd18fa416d9da3bd8f70d33ebb85d39` from BF16 source storage to the
resident F32 artifact contract. Expert tensors remain unchanged source data and
are deferred to M4.1-06.

## Correctness and failure evidence

Direct BF16-to-F32 bit conversion covers normal values, signed zero,
subnormals, infinities, signed/payload NaNs, and samples read from the verified
real norm tensors. Streaming round-trip verification compares every converted
F32 element byte-for-byte without full-tensor residency.

Tests reject wrong complete-shard hash, shard truncation, wrong source dtype,
wrong pinned shape, invalid source range, insufficient disk space, expert
selection, incomplete full inventory, and injected incomplete output. The
injected commit failure leaves the output directory empty.

## Reproduction

```powershell
cargo run --release -p clr-qwen3-moe --example convert_pinned_dense -- complete `
  models\qwen3-30b-a3b\dense-source-plan-v1.tsv `
  <source-root> <output-directory> <available-bytes>
```

`source-root` must contain all 16 complete pinned shards. The converter verifies
their recorded lengths and hashes before decoding any tensor payload.

## Limitations

- Large source shards and converted artifacts are local evidence and are not
  committed.
- BF16 is decoded to F32; no BF16 runtime kernel is introduced.
- Expert conversion, tokenizer parsing, quantization, mmap, SIMD, FFI, and
  optimized kernels remain outside this task.
