# M4.3-02 Candidate Expert Quantization Format

## Result

M4.3-02 defines the first quantization candidate without changing runtime
behavior or creating a quantized model artifact. The selected format is
symmetric INT8 per output channel with F32 scales and F32 accumulation.

The selected format is approved for a future correctness-first implementation.
It is not yet implemented, and its provisional gates are not part of the
frozen F32 registry.

## Candidate Comparison

| Candidate | Max weighted error | Max down error | Per-expert bytes | Full 6,144 experts | Compression | Decision |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| INT8 per tensor | `0.4204293` | `1.3565183` | `4,719,136` | `28,994,371,712` | `4.212807x` | Rejected |
| INT8 per output channel | `0.1043549` | `0.3367004` | `4,733,280` | `29,081,272,448` | `4.200218x` | Selected |
| INT8 input group 128 | `0.0820999` | `0.2648926` | `4,866,400` | `29,899,161,728` | `4.085321x` | Promising; more evidence required |

The per-output-channel candidate reduces maximum weighted error by about 75%
versus per-tensor quantization while adding only row scale metadata. Group-128
is numerically better, but costs 2.81% more bytes than per-output-channel and
requires more indexing and scale-boundary logic. It remains the next comparison
candidate rather than being silently promoted.

Representative maximum projection errors were:

| Candidate | Gate | Up | Activated gate | Product | Down | Weighted output |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Per tensor | `0.1140000` | `0.0897568` | `0.0908598` | `0.3838177` | `1.3565183` | `0.4204293` |
| Per output channel | `0.0346689` | `0.0501070` | `0.0381253` | `0.2740030` | `0.3367004` | `0.1043549` |
| Group 128 | `0.0228627` | `0.0245413` | `0.0210975` | `0.1241150` | `0.2648926` | `0.0820999` |

All three candidates used F32 source weights, INT8 payloads, F32 scales,
nearest-even rounding, saturation to `[-127,127]`, zero-scale handling, and
little-endian deterministic serialization. No NaN or infinity source values
were accepted. Matrix ranges, scale distributions, saturation counts, row
worst cases, reconstruction errors, and cosine similarities are in the
machine-readable evidence.

## Runtime Contract Draft

The first implementation should reconstruct one complete selected projection
into a bounded F32 buffer and call the existing scalar F32 matrix operation.
Activations, routing weights, RMSNorm, residuals, and accumulation remain F32.
The largest temporary projection buffer is `6,291,456` bytes. INT8 matmul,
SIMD, threading, mmap, prefetch, and cache changes are explicitly deferred.

This path favors debuggability: every dequantized projection can be compared
against the source F32 matrix before any kernel optimization. The full draft is
`models/qwen3-30b-a3b/m4.3-02-runtime-kernel-contract-v1.json`.

## Artifact Schema Draft

The additive schema ID is `colibri-qwen3-moe-expert-int8-v1`. It keeps the F32
artifact untouched and records source identity, layer/expert IDs, gate/up/down
descriptors, dimensions, axis/group parameters, aligned offsets, logical
lengths, and component/file SHA-256 values. Records are ordered by layer then
expert, with projection order gate/up/down. The schema is documented in
`m4.3-02-quantized-expert-artifact-schema-v1.json`.

## Provisional Correctness Gates

For the selected candidate, representative stop gates are matrix reconstruction
`0.0078125`, gate `0.04`, up `0.06`, activated gate `0.05`, product `0.30`,
down `0.40`, and weighted expert output `0.13`. These are characterization
limits for the selected cases only. A future implementation must additionally
preserve exact structure, finite outputs, F32-safe router IDs, Tier B top-k
agreement, and Tier A generated IDs `[1096, 374]`.

The gates are provisional and must be recharacterized before promotion. They do
not loosen `m4.2-tolerance-contract-registry-v1.json` or the M4.3-01 F32
baseline.

## Rejected and Deferred Alternatives

Per-tensor INT8 is rejected because its down and weighted errors are materially
higher. Group-128 is promising and may be reconsidered after broader expert and
full-fixture evidence. Q4, quantized matmul, SIMD, threading, mmap, prefetch,
GPU, and FFI work are out of scope for M4.3-02.
