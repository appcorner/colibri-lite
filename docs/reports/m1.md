## Stop-condition record

Two stop conditions were raised and reviewed during M1:

1. A draft assumed `rope_theta = 1000000.0`, conflicting with frozen
   `config.json` value `10000.0`. The resolution made theta config-driven,
   generated the Rust test constant from the fixture, and added two-theta plus
   oracle tests.
2. The stable MSVC `rustc` component became inconsistent. Work stopped before
   force reinstall; the user repaired the toolchain, after which all commands
   passed.

No unverified code from either incident was committed as complete.

- Batch-one, F32-only correctness path.
- Causal attention starts at position zero and has no padding or sliding window.
- No KV cache, tokenizer, generation, model-file loading, or expert residency.
- Scalar kernels prioritize auditability, not performance.
- Test adapters are pinned to the frozen fixture hashes and offsets.
