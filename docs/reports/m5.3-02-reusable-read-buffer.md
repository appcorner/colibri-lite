# M5.3-02 Reusable aligned read-buffer prototype

## Decision

The reusable aligned staging-buffer path is classified
`microbenchmark_only_value`. It is byte-correct and removes one allocation per
expert miss, but it adds a payload copy from the reusable staging buffer into
the cache-owned `Arc`. In the representative full-model matrix it did not
produce a reliable end-to-end benefit and was slower in 9 of 12 matched
comparisons.

Keep `reference_allocated` as the default reader. Keep the reusable path
feature-gated for diagnostic experiments; do not promote it to the production
default in this task. The exact next task is `M5.3-03 Compute profiling`. No
persistent handles, mmap, coalescing, or other optimization is started here.

## Reader and ownership design

The existing reader remains the reference path. Feature
`m5-3-reusable-buffer` adds `ReaderMode::{Reference,ReusableAlignedBuffer}`
and `ArtifactReader::open_with_mode`. `ExpertStore` uses the reusable path only
when that mode is selected; the default `ArtifactReader::open` behavior is
unchanged.

For a miss, the reusable path locks one synchronous `ReusableReadBuffer`,
grows it to the requested length without shrinking, reads the exact range,
verifies SHA-256, copies verified bytes into an owned `Arc<[u8]>` while the
staging lock is held, and then releases the buffer. The cache lease owns the
`Arc`; no borrowed staging slice escapes the load. The mutex serializes the
current synchronous path. There is no hidden pool, unbounded concurrent
growth, unsafe code, or new dependency.

The artifact manifest confirms the fixed Qwen3 expert payload is exactly
`18,874,368` bytes. The safe decoder consumes byte slices and reconstructs F32
values with explicit little-endian conversion, so its required alignment is
one byte. The implementation checks that contract without raw pointer casts.
`Vec<u8>` length is the logical payload length and capacity is retained for
later variable-size reads. New capacity is zero-initialized by safe `resize`;
smaller reads overwrite only their logical range. Failed reads release the
file and staging lock, and the next read can recover.

## Microbenchmark evidence

The release benchmark used the canonical layer-47 plan and authoritative
64-request miss subset. Every payload was SHA-256 checked and filesystem cache
state was uncontrolled.

| ExpertStore path | Mean wall | Mean expert-load | Mean read | Mean hash | Buffer allocations | Reuses | Allocated staging bytes |
|---|---:|---:|---:|---:|---:|---:|---:|
| Reference allocated | 5.800 s | 5.733 s | 0.402 s | 4.969 s | 64 | 0 | 1,207,959,552 |
| Reusable aligned buffer | 5.589 s | 5.527 s | 0.277 s | 4.949 s | 1 | 63 | 18,874,368 |

The reusable path reduced the isolated read interval by approximately 31% and
eliminated 63 allocations. It did not reduce logical payload-copy volume: it
read 1,207,959,552 bytes and copied the same logical payload into cache-owned
storage. The reference path transfers its allocated `Vec` into cache-owned
storage; the reusable path must perform the staging-to-owned copy. This is
allocation elimination, not a general latency claim.

Both paths performed 64 opens, 64 seeks, 64 read calls, 64 misses, and 63
evictions. Reusable counters were one allocation, 63 reuses, zero fallback
allocations, zero alignment failures, and an 18,874,368-byte peak buffer.

## Interpretation and decisions

1. One staging allocation replaces one allocation per miss. The 64-request
   benchmark changed 64 allocations to 1; full runtime changed `loads`
   allocations to 1 per run.
2. Copy volume is not reduced. The reusable path adds an ownership copy after
   verification; logical requested, returned, and cache-owned bytes are
   unchanged.
3. No decoding or construction improvement from alignment was isolated; the
   current safe byte decoder requires only byte alignment.
4. The isolated read improvement did not reproduce end-to-end. Long-context
   and code runs showed significant regressions, consistent with added copy
   cost and uncontrolled timing.
5. Miss-heavy workloads expose the allocation savings in microbenchmarks and
   the copy cost most strongly in full-model execution.
6. The larger cache reduces misses and expert reads for cacheable workloads,
   but does not remove the reusable path’s measured regressions.
7. Retain the reference allocated reader as default. Reusable remains explicit
   and diagnostic only.
8. Persistent handles are not justified by M5.3-01’s small open/close signal
   relative to hashing and payload processing.
9. Select compute profiling next because scalar matrix-compute time was not
   isolated in M5.3-01 and this storage prototype has no generalizable
   end-to-end value. Do not start it in this task.

The configurable expert-cache memory primitive remains accepted with workload
limitations. This reader prototype is not accepted as a production storage
primitive; its diagnostic value is separating allocation savings from the
ownership-copy cost.

## Limitations

uncontrolled filesystem-cache state. No token/s, cold-cache, physical-I/O, or
linear throughput claim is made. Short English and repeated-pattern remain
simulation-only in this task. A single staging buffer is safe for the current
synchronous path; concurrent/nested loads are intentionally not introduced.
