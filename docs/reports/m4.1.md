## Acceptance evidence

- Exact source identity, architecture, license, config, and tensor inventory
  are pinned and validated.
- Dense and expert BF16 source payloads were converted deterministically to F32
  with bounded explicit buffers and exact round-trip verification.
- Every dense tensor, expert tensor, logical expert, expert shard, and tokenizer
  asset is accounted for.
- The root manifest was generated twice with byte-identical output.
- Metadata validation passed before and after moving the complete directory.
- Full integrity validation hashed all 122,147,678,312 artifact bytes including
  the root manifest and passed.
- Failure tests cover missing, renamed, truncated, corrupted, duplicate,
  incomplete, unsupported, incompatible, and unknown-critical-field cases.
- All five standard Cargo verification commands pass with 118 workspace tests,
  zero failures or ignored tests, no Clippy warnings, and a successful CLI
  smoke run.

## Remaining limitations

- Rust inference still accepts token IDs; text tokenization remains an offline
  reference adapter.
- The chat template is preserved but not rendered.
- The artifact is F32 and requires approximately 113.76 GiB of storage.
- No quantization, mmap, SIMD, GPU support, or performance optimization exists.
- Full-model numerical inference has not yet been validated; that begins at
  M4.2-01.
