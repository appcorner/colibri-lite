# M5.1-03 Full-Model Expert-Cache Validation

## Result

The configurable F32 strict-global-LRU expert cache was executed against the
canonical Qwen3-30B-A3B artifact. Baseline, exact 8 GiB, and exact 16 GiB
payload budgets preserved frozen generation `[1096, 374]`, retained F32
checkpoints, deterministic routing, finite outputs, KV-cache invariants, and
bounded payload residency. Classification remains `accepted_with_limitations`.

Artifact validation read all 57 manifest-listed payload files (122,147,666,917
bytes); root SHA-256 is
`f133d733612840ad691d637732d4ef2de1e0242c4bb1d92521b49dfcfb1b8cd2`.

## Fixture and Command

The exact replay used input IDs `[9707, 11, 1879, 0]`, six positions including
the two generated IDs, 2,304 expert occurrences, KV capacity 6, F32 compute,
and strict global LRU. `COLIBRI_ARTIFACT_ROOT` was set to the validated

`clr_qwen3_moe-e1503717729029d2.exe full_model_validation_tests::short_cached_generation_matches_transformers --exact --nocapture`

Each run used `COLIBRI_TRACE_ONLY=1` and one of `18874368`, `8589934592`, or
`17179869184` for `COLIBRI_EXPERT_CACHE_BUDGET_BYTES`. Filesystem cache state
was uncontrolled. Full-vocabulary logits were not retained; retained F32
checkpoints and invariant assertions were used.

Exact-budget runtime counters match an independent exact-budget replay of the
authoritative ordered trace. M5.1-02 reported `719/1585/1150` and
`929/1375/485` because it used usable payload budgets of 8,212,550,504 and
16,802,485,096 bytes after modeled overhead. This is an accounting difference,
not a cache-policy or execution-order mismatch.

Expert logical reads fell 31.21% at 8 GiB and 40.41% at 16 GiB; total logical
reads fell 18.59% and 24.07%. Relative to the single-sample baseline run,
prefill improved 16.11%/26.43%, decode 48.63%/82.23%, and total wall time
25.14%/40.46%. These timings are directional only: filesystem cache was
uncontrolled, the harness performs retained checkpoint validation, and no
physical-device I/O reduction is claimed.

## Memory and Correctness

Peak expert payload stayed below each configured budget. Oversized-entry and
blocked-eviction counts were zero. Modeled explicit peak bytes were 120,529,096
(baseline), 8,689,492,168 (8 GiB), and 17,277,329,608 (16 GiB), including fixed
F32 buffers and KV cache. Process-level working-set/private-byte sampling was
not available from this harness and remains an open measurement limitation.

All runs preserved generated IDs, guard routing and selected-expert ordering,
retained F32 intermediate/final-norm checks, finite outputs, deterministic
execution, fixed KV-cache allocation/length, and bounded memory. No arithmetic,
artifact, dense path, or cache policy changed.

## Decision

Classification remains `accepted_with_limitations`. Exact 8 GiB is the
recommended balanced payload-cache operating point; 16 GiB is a useful
higher-memory mode, but its marginal value and memory cost should be evaluated
on broader traces before a permanent preset is added.
