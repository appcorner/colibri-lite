# M5.1-02 Configurable Expert-Cache Prototype

## Scope

This task promotes the existing safe Rust byte-budgeted `ExpertCache` into the
reviewed configurable prototype contract. It remains F32-only, uses strict
deterministic global LRU, and does not change dense storage, arithmetic,
routing, artifact layout, or cache policy.

`ExpertStore::new(reader, registrations, budget_bytes)` is the canonical API.
The budget is an explicit payload-byte value. Metadata and allocator/alignment
effects are not charged against the payload budget; they are reported by the
replay evidence. The default one-expert budget used by the frozen baseline is
unchanged.

## Implementation

`clr-storage::ExpertCache` now exposes configured budget and expanded metrics:
resident/peak entry counts, bytes served from cache, bytes avoided, oversized
entry events, and blocked-eviction events. LRU recency remains a monotonic
deterministic clock with `(last_used, ExpertKey)` tie-breaking. Active leases
pin entries through `Arc`; an oversized payload is rejected and a request that
cannot evict unleased entries returns `CacheBudgetExhausted`.

## Trace replay

The authoritative M5.1-00 order was adapted to a temporary TSV and replayed
through the production Rust cache. This is an input adapter only; order was
not reconstructed and no model arithmetic ran. The replay matches the
simulation counters exactly:

| Modeled RAM | Payload budget | Hits | Loads | Evictions | Expert bytes avoided | Total modeled reads |
|---:|---:|---:|---:|---:|---:|---:|
| 8 GiB | 8,212,550,504 | 719 | 1,585 | 1,150 | 13,570,670,592 | 59,434,164,224 |
| 16 GiB | 16,802,485,096 | 929 | 1,375 | 485 | 17,534,287,872 | 55,470,546,944 |

The replay uses the usable expert-cache budgets from the M5.1-01 total-RAM
scenarios so the simulation's metadata/alignment accounting remains directly
comparable. Runtime payload residency stays within the configured payload
budget. Peak-byte differences are exactly the modeled metadata/alignment
charge, not policy differences.

## Full-model validation limitation

Full-model baseline/8 GiB/16 GiB correctness and timing runs were not executed:
this checkout has no canonical dense/expert payload directories and no
`COLIBRI_ARTIFACT_ROOT`. No timing, throughput, or physical-I/O claim is made.
Frozen F32 IDs `[1096, 374]` and all M4 contracts remain unchanged by the
cache-only code path.

## Decision

Classification: `accepted_with_limitations`. The memory primitive is accepted
for further controlled validation because actual LRU counters and bounded
payload residency match M5.1-01. Full-model correctness and latency remain
open gates. The recommended operating point is the modeled 8 GiB balanced
configuration until a broader trace corpus and canonical replay environment
are available.

Next candidate: broader representative expert traces. No resident-dense,
mmap, prefetch, SIMD, threading, quantization, or GPU work starts here.
