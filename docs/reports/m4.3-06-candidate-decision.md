# M4.3-06 Candidate Decision

## Formal decision

The candidate `symmetric INT8 per output channel with F32 scales, F32
activations, and F32 accumulation` is rejected as a full-model production
candidate. It is retained as a diagnostic/reference format and may be studied
for selective layers or projections only after new evidence. No quantized
runtime, ExpertStore path, cache capacity, artifact, router, or arithmetic was
changed.

Selected statuses are `rejected_full_model_candidate`,
`retained_for_diagnostics`, and `candidate_selective_use_only`.

## Evidence

The format gives approximately `4.200218x` expert compression: `4,733,280`
bytes per expert and an estimated `29,081,272,448` bytes for all 6,144
experts. It supports 226 to 7,259 resident experts under 1 to 32 binary GiB
budgets. Those storage results are attractive but do not compensate for the
quality-risk propagation.

Local Tier-C diagnostics passed the provisional representative gates. Maximum
local down-projection error was `0.3367004`; maximum weighted expert-output
error was `0.1043549`. Full-depth Tier-B propagation first became materially
risky at Layer 1 (`0.1724930` final-block error) and reached its largest final
block drift at Layer 47 (`152.1300659`). This is accumulated hidden-state drift
amplified by later expert/router operations, not evidence of an F32 defect.

The Thai Tier-B fixture changed argmax from `7360` to `16222` under a
numerically ambiguous margin. Minimum Top-20 overlap was `0.65`; maximum rank
displacement was `13`. Tier A still generated `[1096, 374]`, but several
generation steps were ambiguous and several guard router IDs were not
exact-safe. Matching those two IDs alone is therefore insufficient for
acceptance. The non-timing evidence was deterministic and byte-identical.

Per-tensor INT8 is rejected from its larger local down (`1.3565183`) and
weighted (`0.4204293`) errors. Group-128 is promising (`0.0820999` maximum
weighted local error) but requires full propagation evidence and costs 2.81%
more bytes. It is not promoted by this decision.

## M4.3 phase verdict

| Area | Final status | Decision |
| --- | --- | --- |
| Ordered Rust F32 baseline | authoritative and accepted | frozen reference for all future variants |
| INT8 per tensor | rejected | do not implement |
| INT8 per output channel | rejected for full-model production | retain diagnostics/selective investigation only |
| INT8 group-128 | promising but insufficient evidence | requires new full-depth evidence |
| Router INT8 | rejected | router remains F32 |
| Dense BF16 candidates | candidate requiring kernel/activation evidence | storage-only observation is not acceptance |
| ik_llama Q4_K_M | external performance reference only | no quality-equivalence claim |

Remaining quantization uncertainty concerns group-wise error after long
propagation, activation/kernel precision, calibration or layer-selective
formats, and model-quality metrics beyond the compact fixtures.

## External reference interpretation

The pinned CPU-only Q4_K_M ik_llama run measured approximately `3.968` prompt
tok/s, `2.819` long-run decode tok/s, and `5.276-5.863` short-run decode tok/s,
with `4.36-9.03 GB` sampled peak working set. The comparison is directional:
Q4_K_M, mmap/page cache, optimized kernels, SIMD, threading, fused MoE, and
graph reuse differ from colibri F32. The hardware is capable of useful speed,
but the gap cannot be attributed to quantization alone. ik_llama quality was
not established as equivalent to the frozen F32 baseline.

## Investment decision

Continue the project, but pivot from immediate quantized-runtime implementation
to memory hierarchy and compute-path work. Immediate production INT8 should not
proceed. Colibri's low-memory differentiation remains valuable; the current
correctness path is memory-efficient but not performance-ready.

## Frozen invariants

Future work must preserve canonical artifact hashes/layout, ordered F32 and
RMSNorm contracts, deterministic router tie ordering, Tier-A IDs `[1096, 374]`,
F32-safe guard-layer IDs, selected intermediate evidence, finite outputs,
fixed-capacity KV-cache allocation/no-overwrite behavior, bounded memory, and
evidence-schema compatibility. Speed alone cannot accept a change.

## Next optimization experiment

The single recommended first experiment is:

`F32 memory-hierarchy study: resident dense weights plus trace-driven
expert-cache sizing`.

The simulation must use recorded request traces and model separate dense-
resident and expert-cache configurations at 1, 2, 4, 8, 16, 24, and 32 binary
GiB. It reports predicted hits, misses, evictions, bytes avoided, and RAM
requirements without changing runtime code. A prototype is considered only
after simulation review. Initial gates are frozen F32 correctness, materially
lower modeled logical reads per token, measurable bounded cache hits, no
artifact duplication, and throughput measurement only after a reviewed
memory-path prototype exists.
