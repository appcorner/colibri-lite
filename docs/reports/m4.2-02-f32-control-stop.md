# M4.2-02 Layer-0 F32 Control Review Stop

## Resolution

The focused ordered-scalar diagnosis subsequently proved that Rust RMSNorm is
correct under its deterministic F32 arithmetic contract. The mismatch recorded
below is now classified as cross-runtime reduction-order variance, not an
implementation defect. ADR 0020 defines the approved provisional validation
budget, and `m4.2-02-layer0-router-validation.md` records the resumed layer-0
router result. This document remains the historical evidence for the original
review stop.

## Decision result

The approved Transformers F32 control used the same pinned BF16 source bytes,
decoded those exact values to F32, and executed the same layer-0 inputs,
positions, causal mask, eager attention, norms, residuals, and router path. It
did not load a different model or execute experts.

Transformers F32 and Rust F32 first diverge outside the existing F32
correctness contract at `post_attention_rmsnorm`. This is therefore classified
as an implementation mismatch for review, not solely BF16 precision
amplification. Router F32-versus-Rust comparison, expert-ID assertion, later
layers, and expert execution were not performed after this stop.
