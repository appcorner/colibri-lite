## Scope review

- No external Rust dependency was added.
- No unsafe code, memory mapping, SIMD, FFI, quantization, generation, KV cache,
  server, GPU backend, or additional architecture was added.
- One logical expert may occupy a range inside a shard; one-file-per-expert is
  not part of the contract.
- Resident M1 APIs and correctness oracle remain unchanged.

## Known limitations

- Manifest serialization is deferred to the M4 converter.
- Expert payloads are F32 and decoded into temporary vectors.
- File open/read/hash occurs on cache miss.
- Full-model decode profiling has not yet been performed.
- Memory mapping is deferred under explicit reconsideration criteria.

## Milestone decision

All M2 correctness, residency, budget, corruption, equivalence, metrics, and
portable-backend criteria pass. M2 is closed without memory mapping. Work may
proceed to M3 on `milestone/m3-generation`, beginning with M3-01.
