## Instrumentation

`ExpertStore::load_with_observer` delegates to the existing `load` method,
then reports metric deltas. The Qwen streaming diagnostic callback records
the request after the lease is acquired and before expert computation. The
existing expert iteration order, router output, cache behavior, and payload
decode are unchanged. Records are accumulated in memory and serialized only
after the generation completes.

Each record contains the contiguous global ordinal, prefill/decode phase,
generation step, absolute position and input token, layer, selected rank,
expert ID and stable key, payload bytes, cache-hit/load flags, and evictions.
The embedded header records the baseline, release tag, source and
instrumentation commits, model/artifact identity, cache/runtime configuration,
and deterministic serialization rules.

## Correctness Replay

Both the canonical replay and an independent repeat completed successfully.
They preserved generated IDs `[1096, 374]`, all guard-layer router IDs,
finite outputs, fixed KV-cache allocation/length/no-overwrite invariants, and
the selected intermediate and final checkpoint contracts. Both runs reported
the frozen dense and expert logical-read counters.

The temporary full-vocabulary logits used by the original M4 measurement were
not retained in the repository. The replay therefore uses the committed
fixed/top-20 checkpoints for trace-only selection validation and explicitly
does not claim a new full-vocabulary logit comparison. This limitation does
not affect request ordering or the frozen generated-ID/guard/intermediate
checks.
