# M4.2-01 Selected Full-Model Tensor Value Validation

## Scope

Validated selected source BF16 values from pinned Qwen3-30B-A3B Safetensors
against the stable F32 dense and expert artifacts. This task performed no
Transformer execution, routing, inference, conversion, or optimization.

Source identity:

- Model: `Qwen/Qwen3-30B-A3B`
- Revision: `ad44e777bcd18fa416d9da3bd8f70d33ebb85d39`
- Stable artifact root:
  `D:\models\colibri-lite\qwen3-30b-a3b\artifact-v1`
- Root-manifest SHA-256:
  `f133d733612840ad691d637732d4ef2de1e0242c4bb1d92521b49dfcfb1b8cd2`

## Selection

The deterministic selection covers 13 dense tensors:

- untied token embeddings and LM head;
- final and layer-0 input normalization;
- router gates at layers 0, 24, and 47;
- Q/K/V/O attention projections at layer 0;
- Q projections at layers 24 and 47.

It also covers gate/up/down projections for experts `(layer 0, expert 0)`,
`(layer 24, expert 64)`, and `(layer 47, expert 127)`. Each tensor uses the
first, midpoint, last, and a deterministic name-hash element coordinate,
de-duplicated when coordinates coincide.

## Validation

For each selected tensor, the validator checks plan/manifest shape, source
shard, source offset, and source/artifact byte lengths. It hashes every selected
source shard against pinned provenance before reading values. Each source BF16
word is decoded by shifting its bits left by 16 and must equal the stored
little-endian F32 word exactly.
