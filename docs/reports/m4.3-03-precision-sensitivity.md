# M4.3-03 Precision Sensitivity and Policy

## Verdict

M4.3-03 defines a conservative non-expert precision policy. No production
runtime, ExpertStore, cache, artifact, or arithmetic changed. The canonical
F32 artifact remains read-only and is the authority for all future variants.

The first implementation policy is:

- router weights, both RMSNorm groups, Q/K normalization, final RMSNorm,
  routing weights, residuals, activations, and accumulations remain F32;
- embedding, attention Q/K/V/O, and LM-head weights are future `candidate_bf16`
  storage candidates with F32 compute, not accepted mixed-precision behavior;
- dense INT8 is diagnostic-only; INT8 router is rejected after a safe-margin
  mismatch.

## Scope and Evidence

The deterministic evaluator
`python/reference/evaluate_m4_3_03_precision_sensitivity.py` read selected
canonical dense tensors for Layers 0, 1, 24, and 47, embedding/LM-head rows for
token IDs `[0, 11, 9707, 125451, 151643]`, and four frozen expert-input vectors
from M4.2 intermediate evidence. It measured F32, BF16-rounded-to-F32, and
offline per-output-channel INT8 reconstructions. There are 117 records and 12
router records. No mixed artifact or optimized kernel was created.

Because the canonical F32 artifact was converted from BF16, BF16 rounding of
weights is exact (`0.0` maximum error for every measured group). This is a
controlled weight-storage observation, not evidence for BF16 activations,
reduction trees, or kernels.

The Layer-0 safe-boundary mismatch is an immediate rejection under the M4.2
router contract. BF16-rounded router weights were bit-identical in this
weight-only experiment; router precision remains F32 because actual reduced
precision router arithmetic was not evaluated and the decision tensor is
semantically critical.

## Mixed-Precision Contract

`models/qwen3-30b-a3b/m4.3-03-mixed-precision-policy-v1.json` is a draft, not
the F32 contract registry. Any future candidate must keep F32 activations,
routing weights, residuals, softmax, RMSNorm arithmetic, and accumulation until
separate evidence authorizes a change. Acceptance requires Tier C diagnostics,
all six Tier B fixtures, and Tier A generated IDs `[1096, 374]`, plus finite
outputs, unchanged router IDs/safe margins, top-k vocabulary agreement, and
fixed KV-cache behavior.

## Limitations and Rejected Cases

- No complete mixed-precision forward was run; propagated hidden-state and
  generation effects remain unmeasured.
- LM-head evidence uses deterministic selected rows, not a full vocabulary
  top-20 comparison. Full vocabulary is a required future gate.
- BF16 tests rounded weights only; they did not change activation dtype or
  reduction order.
- INT8 attention/router results are offline dequantized F32 diagnostics, not
  optimized kernels. INT8 router is rejected; other INT8 dense groups remain
  diagnostic-only.
- Storage estimates are logical bytes and do not claim RAM or physical-I/O
  savings until an implementation is measured.
