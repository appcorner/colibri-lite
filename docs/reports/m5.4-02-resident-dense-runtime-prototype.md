# M5.4-02 Measurement-Only Resident-Dense Runtime Prototype

## Scope and status

M5.4-02 is complete-for-review as an opt-in, measurement-only prototype. It
does not authorize production or default-path adoption. The default streamed
`File` reader, reference execution path, F32 tolerances, artifact format, and
strict global-LRU expert cache are unchanged.

The candidate is available only with
`full-model-validation,m5-4-resident-dense` and the explicit
`COLIBRI_DENSE_RESIDENCY_MODE=resident_dense` option. It loads the canonical
non-expert dense payload once into safe Rust-owned memory; expert payloads
remain on demand in the existing strict global LRU. Without the option, the
same build retains the streamed reader.

## Timing and retained diagnostics

Initialization, prefill, decode, and total wall-time values are present per
row in the machine-readable result. Total-time ranges were 47.78-325.30 s
(8 GiB streamed), 55.57-449.77 s (8 GiB resident), 55.15-299.63 s
(16 GiB streamed), and 57.54-436.40 s (16 GiB resident). These single-run,
uncontrolled measurements are directional only and support no speed, latency,
or throughput claim.

`e263455bb46e865b5093f046f8f4b94fc872e24caba9304ded8fc1efa250f649`) records

allocation, numerical, router, KV, or cache-policy failure.

The complete machine-readable source of truth is
`models/qwen3-30b-a3b/m5.4-02-resident-dense-runtime-results-v1.json`.

## Decision

Classification: `prototype_insufficient_runtime_value`.

The prototype provides reproducible correctness and memory/cache evidence, but
the directional timing and unmeasured physical-I/O/page-cache behavior do not
justify a runtime-value or production-performance conclusion. M5.4-02 remains
measurement-only; production/default adoption is not authorized.
