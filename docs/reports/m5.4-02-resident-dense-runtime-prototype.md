# M5.4-02 Measurement-Only Resident-Dense Runtime Prototype

## Scope and status

This is an opt-in, test-only measurement prototype. The default dense reader,
reference execution path, artifact format, F32 tolerances, and strict global
LRU expert cache are unchanged. No production adoption or performance decision
is authorized by this task.

The prototype is compiled only with `full-model-validation,m5-4-resident-dense`.
`COLIBRI_DENSE_RESIDENCY_MODE=resident_dense` loads the canonical dense payload
once into safe Rust-owned memory. Without that explicit option, the same build
uses the existing streamed `File` reader. Experts remain on-demand through the
existing `ExpertStore` and strict global LRU.

## Budget accounting

The candidate rejects its configuration before loading dense payload when:

```text
total RAM budget >= resident dense weights + fixed runtime memory + expert cache
```

`fixed runtime memory` is `377,384,088` bytes. It retains the M5.4-01
runtime/safety allowance and adds the observed peak dense tensor decode buffer,
so it is deliberately stricter than the simulation. The configured candidate
expert cache is `2,048,030,150` bytes at 8 GiB and `10,637,964,742` bytes at
16 GiB. Cache admission remains the existing strict global LRU implementation.

Logical tensor access is reported separately from the dense payload read during
initialization. Neither is a physical-device-I/O measurement; Windows page
cache and storage-device activity remain unmeasured. The existing Windows
working-set sampler is retained for an actual capture.

## Provenance and fixtures

The prototype is pinned to Qwen/Qwen3-30B-A3B revision
`ad44e777bcd18fa416d9da3bd8f70d33ebb85d39`, F32 artifact format v1, canonical
root hash `f133d733612840ad691d637732d4ef2de1e0242c4bb1d92521b49dfcfb1b8cd2`.
It uses the six M5.2 fixtures with recorded runtime dense-read evidence:
`tier_a_control`, `tier_b_short_thai`, `tier_b_special_token`,
`tier_b_code_newline`, `long_english_context`, and `long_decode_english`.

`tier_b_short_english` and `tier_b_repeated_pattern` remain unavailable for
this matrix because M5.2-03 never captured their full-runtime dense-read
evidence. They were not silently omitted.

## Correctness and measurements

Focused tests validate resident loading, range bounds, over-budget rejection,
and file release. The full-model harness compiles with the prototype feature;
its existing F32 checkpoints, exact tokens/router order, finite-output, KV,
and cache assertions remain intact.

No full-model capture was executed: the canonical 122 GB artifact root is not
mounted in this workspace. No model was downloaded, copied, or synthesized,
and the 122 GB capture was not rerun. Therefore there are no candidate timing,
working-set, cache, logical-read, or correctness claims beyond the focused
tests. The result JSON records these values as unavailable rather than
inventing measurements.

## Decision

Classification: `prototype_insufficient_runtime_value`.

The implementation is ready for a separately controlled capture on the
registered canonical artifact, but no runtime result exists here to establish
value. This remains measurement-only and is not a production/default path.
