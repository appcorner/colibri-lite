# M4.1-06 Expert Conversion Report

## Scope

Converted every expert tensor from pinned `Qwen/Qwen3-30B-A3B` revision
`ad44e777bcd18fa416d9da3bd8f70d33ebb85d39` from BF16 source storage to the
existing packed F32 expert contract. The runtime still loads one logical expert
by layer and expert index through `ExpertStore`; the shared gate/up/down
numerical path is unchanged. The pinned upstream license is Apache-2.0.

## Artifact layout

The complete artifact contains 48 physical layer containers. Each container
holds 128 consecutive logical expert ranges, not individual tensor files. The
manifest can map any expert to an arbitrary shard ID and offset.

Packing order is layer ascending, expert ascending, then gate, up, and down.
Gate and up shapes are `[768, 2048]`; down shape is `[2048, 768]`. The pinned
source orientation matches the M1 math, so no transpose or layout transform is
performed.

## Correctness and failure evidence

Each source shard is length- and hash-verified before any payload decode.
Conversion and exact round-trip verification stream fixed-size chunks and never
retain a full expert or layer. The unchanged `ArtifactReader` and `ExpertStore`
loaded expert `23:64` by its logical range and reported exactly 18,874,368
bytes read, rather than the 2,415,919,104-byte layer container.

Tests reject source corruption, truncation, invalid offsets, wrong shard,
wrong layer/expert identity, wrong shape, incomplete inventory, duplicate
experts, invalid gate/up/down order, a one-layer vertical slice, and injected
incomplete output. Invalid chunk sizes and insufficient preflight space also
fail with structured errors before source or output access. The failed
transaction cleanup test leaves no output directory.

## Reproduction

```powershell
cargo run --release -p clr-qwen3-moe --example convert_pinned_experts -- complete `
  models\qwen3-30b-a3b\expert-source-plan-v1.tsv `
  <source-root> <output-directory> <available-bytes> -
```

The evidence runs used Rust/Cargo 1.96.1 on the
`x86_64-pc-windows-msvc` host toolchain and Windows NT 10.0.26200.0.

## Limitations

- Large source shards and converted expert containers remain local and are not
  committed.
- No quantization, BF16 kernel, mmap, async prefetch, SIMD, FFI, tokenizer
  parsing, optimized kernel, or full-model generation is included.
- This is a correctness artifact, not a performance result.
