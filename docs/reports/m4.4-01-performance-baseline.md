# M4.4-01 Versioned Performance Baseline

## Decision

The canonical baseline index is
`models/qwen3-30b-a3b/m4.4-performance-baseline-v1.json`.
Its stable identifier is `qwen3-30b-a3b-colibri-f32-windows-x64-v1` and its
schema is `colibri-qwen3-moe-m4.4-performance-baseline-v1`.
The index references existing evidence by canonical relative path, byte count,
and SHA-256. It does not contain model payloads or duplicate large tensors.

The payload is deterministic: UTF-8 JSON, sorted keys, compact separators,
one trailing newline, and no timestamp or newly measured wall-clock values.
The generated index SHA-256 is
`29b2d95fa9eb74c1085cb31d2f63adbaa711fe8739d3051fa04f7b2b1c27ce9d`.

## Contents

The index records model and artifact identity, runtime/build identity, the
accepted M4.2/M4.3 F32 correctness evidence, Tier A/B/C fixtures, generated
IDs `[1096, 374]`, guard-layer routing, selected expert intermediates, final
norm/LM-head and KV-cache contracts, the M4.2-05 logical resource baseline,
and the directional ik_llama Q4_K_M reference.

It also records the final M4.3 quantization statuses, frozen optimization
invariants, the 1/2/4/8/16/24/32 GiB trace-study inputs, the future comparison
record fields, and M5 throughput and memory-hierarchy gates.

## Validation

Generation fails if a referenced file is missing, a reference hash or byte
count changes, a known evidence schema/version is incompatible, the canonical
Qwen3 revision or artifact inventory changes, the F32 baseline is not
authoritative, cache/read counters do not reconcile, the frozen fixture changes,
or a non-F32 quantization candidate is marked production-accepted. Baseline IDs
must be unique within the model index directory.

The focused test suite verifies required sections, all 31 references and their
hashes, frozen model/performance/status values, and byte-identical repeated
generation. No runtime, artifact, cache capacity, or numerical behavior was
changed. No RAM/cache simulation was run.

## Limitations

Performance values are inherited from M4.2-05 and are application-level logical
reads, not physical device I/O. The filesystem cache was uncontrolled and the
baseline makes no cold-cache claim. The ik_llama comparison remains directional
because its Q4_K_M model, mmap, SIMD, threading, fused MoE, and graph reuse are
not format-controlled against the colibri F32 runtime; it is not a quality
equivalence claim.
