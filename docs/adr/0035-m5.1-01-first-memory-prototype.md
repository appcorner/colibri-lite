# ADR 0035: Select the first M5 memory-hierarchy prototype

## Status

Accepted for planning; implementation deferred to the next reviewed task.

## Context

M5.1-01 replays the authoritative ordered M5.1-00 trace from the frozen F32
baseline. The trace contains 2,304 expert occurrences, 1,332 unique
layer/expert keys, 43,486,543,872 expert logical-read bytes, and reuse
distances of 379/384/1,924 (minimum/median/maximum). The simulator models
binary-GiB budgets, explicit runtime overhead, 64-byte cache metadata, and
4096-byte alignment. It does not change Rust execution or claim throughput.

## Decision

The first runtime prototype will be a **configurable larger expert cache**.
It will be evaluated only after implementation-specific correctness and
bounded-memory gates are approved. Resident dense weights are not selected as
the first prototype.

## Evidence

- Global LRU requires 379 charged entries (`7,154,937,856` bytes) for the
  first hit. The first fixed total-RAM point with a hit is 8 GiB.
- At 8 GiB streamed-dense LRU, expert-byte hit rate is 31.21% and modeled
  total logical reads fall 18.59%.
- At 16 GiB streamed-dense LRU, hit rate is 40.32%; at 24 GiB it is 42.19%.
- Full unique-key residency requires 25,140,658,176 payload bytes, or
  25,146,114,048 bytes including modeled entry charge.
- Full dense residency is infeasible at 1/2/4 GiB. At 8 GiB it leaves only
  about 1.98 GiB for experts and produces no LRU hits on this trace; at 16 GiB
  it avoids dense logical reads but retains fewer expert hits than streamed
  dense at the same budget.

## Alternatives

- **Resident dense first:** rejected for isolation. It removes dense logical
  reads but competes directly with the expert working set and has no hit at
  the 8 GiB operating point.
- **Combined resident dense plus expert cache:** deferred until a separate
  dense-residency prototype can preserve the F32 invariants.
- **No prototype:** rejected because the trace demonstrates material,
  deterministic expert-read reduction at bounded budgets.

Belady results are a theoretical offline ceiling only. Layer-aware LRU and
frequency-aware policies remain diagnostics; no production policy is fixed by
this single short trace.

## Constraints for the next task

The prototype must preserve the canonical artifact hash, ordered F32/RMSNorm
contract, deterministic routing and tie policy, Tier-A generated IDs
`[1096, 374]`, guard-layer IDs, selected intermediates, fixed KV-cache
contract, finite outputs, and strict resident-byte accounting. It must report
logical reads separately from physical I/O and must not introduce quantization,
mmap, prefetch, SIMD, threading, or GPU support.
