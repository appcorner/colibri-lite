# ADR 0037: Full-Model Configurable Expert-Cache Validation

## Status

Accepted with limitations.

## Decision

Validate the reviewed strict global-LRU F32 expert cache on the canonical
Qwen3-30B-A3B artifact at the frozen one-expert baseline and exact nominal
8 GiB and 16 GiB payload budgets. Retain it as an accepted memory primitive.
Use 8 GiB as the initial balanced operating point and keep 16 GiB as a measured
higher-memory mode.

## Evidence

All three runs preserved `[1096, 374]`, retained F32 checkpoints, deterministic
routing and expert order, finite outputs, KV-cache invariants, and payload
budget bounds. Exact-budget replay and runtime counters agree:

- 8 GiB: 719 hits, 1,585 loads, 1,130 evictions, 13,570,670,592 bytes avoided.
- 16 GiB: 931 hits, 1,373 loads, 463 evictions, 17,572,036,608 bytes avoided.

The different M5.1-02 counters are expected because that simulation charged
modeled overhead before assigning payload bytes. No physical I/O claim is made.

## Limitations and Next Task

The fixture is short and one timing sample was collected per mode with
uncontrolled filesystem cache. Process working-set sampling and retained full
vocabulary logits were unavailable; retained checkpoint and invariant evidence
is the scope of this supplement. Broader representative traces are required
before choosing permanent presets or further memory-hierarchy work.

Exact next task: **M5.2-01 Capture broader representative expert traces**.
