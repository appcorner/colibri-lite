# ADR 0027: M4.2 Correctness Contract Registry

- Status: Accepted for M4.2 closure
- Date: 2026-07-16
- Milestone: M4.2
- Task: M4.2-06
- Model revision: `ad44e777bcd18fa416d9da3bd8f70d33ebb85d39`

## Context

M4.2 accumulated several kinds of numerical evidence: exact conversion and
structural checks, exact ordered-F32 checks, provisional cross-runtime guards,
layer-specific propagated budgets, selected-case intermediate budgets,
fixture-specific generation budgets, and semantic margin rules. Treating all
of these as one tolerance would hide their different scopes and could allow a
future optimization to weaken a stronger contract accidentally.

The milestone also recorded apparent failures that were diagnosed as PyTorch
reduction-order variance, accumulated incoming drift, or a reference exporter
batch-shape change. Those diagnoses must remain attached to the corresponding
contract instead of being reduced to larger global tolerances.

## Decision

The authoritative machine-readable registry for M4.2 is:

`models/qwen3-30b-a3b/m4.2-tolerance-contract-registry-v1.json`

Every checkpoint record names its comparison paths, exact/tolerant/margin
mode, absolute and relative tolerance where applicable, ULP guard where
applicable, budget formula, scope status, supporting documents, and stop
condition. Evidence references are content-addressed and validated by focused
tests.

The statuses have deliberately different meanings:

- `frozen`: exact internal, structural, or scalar contract;
- `provisional`: measured cross-runtime guard that is not universal;
- `layer_specific`: propagated budget for the named M4.2 layer/fixture;
- `fixture_specific`: selected-case or six-position regression budget;
- `diagnostic_only`: observation that is not a pass tolerance;
- `semantic_margin`: classification rule for discrete selection.

The ordered Python F32 versus Rust RMSNorm contract remains bit-exact. The
PyTorch same-input RMSNorm guard remains provisional at both 8 ULP and `5e-7`;
passing M4.2 does not promote it to a universal runtime tolerance.

ADR 0019 remains authoritative for router ties: higher score first, then lower
expert ID for exact ties. Exact upstream ID assertions additionally require a
safely positive boundary margin. Transformers F32 is the numerical oracle for
Rust F32. Transformers BF16 remains an independent model-behavior reference.

Propagated budgets from ADRs 0020 through 0025 are frozen as regression limits
only for their recorded M4.2 layers and fixture. They are not general kernel
error bounds and cannot be copied to new prompts, platforms, dtypes, backends,
or accumulation algorithms without measurement and review.

## Closure Decision

M4.2 is **completed with documented numerical variance**.

No unresolved implementation defect was found in the validated Rust F32 path.
Known correctness risks are coverage limits: one compact generation fixture,
selected intermediate cases, provisional cross-runtime reduction guards, and
no validation yet for quantized or alternative arithmetic. These are recorded
separately from the current scalar path's performance limitations.

Future optimization work must satisfy
`docs/m4.2-optimization-invariants.md`. A change that cannot preserve an
existing invariant must stop for review and define a separately versioned
contract; it must not edit this registry in place merely to pass.

## Consequences

- Runtime arithmetic, RMSNorm, router ordering, activation semantics, cache
  size, artifact layout, and evidence values do not change in M4.2-06.
- Registry consistency and evidence hashes are tested without model execution.
- The frozen M4.2-04 generation regression remains the end-to-end gate.
- Optimization and M4.3 remain unstarted pending review of this closure.
