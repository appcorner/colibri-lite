# ADR 0045: Resident-Dense plus Strict Global-LRU Candidate Simulation

## Status

Accepted for separate prototype review; runtime implementation is not
authorized by this ADR.

## Context

M5.3 established that the current runtime is storage-bound, but the reusable
buffer and mmap experiments did not produce acceptable end-to-end value. The
remaining candidate is to keep dense weights resident and use the existing
strict global LRU policy for routed experts. This changes the memory trade-off
and therefore must be evaluated before any runtime change.

## Decision

Run a simulation-only candidate study using the validated M5.2 corpus and the
six workloads with recorded full-runtime dense-read evidence. Model total
binary-GiB budgets by reserving the dense artifact, runtime buffers, runtime
structures, and safety reserve before assigning the remainder to strict global
LRU expert payload capacity.

The candidate is selected for a separate measurement-only runtime prototype
review because the simulation shows meaningful modeled logical-read reduction.
This decision does not create a production preset, change the default reader,
change the cache policy, or authorize runtime implementation.

## Evidence

| Total budget | Streamed dense | Resident dense | Resident expert cache |
|---:|---:|---:|---:|
| 8 GiB | 16.31% modeled read reduction | 40.43% | 1.981 GiB; 0% expert-byte hits |
| 16 GiB | 21.36% modeled read reduction | 56.90% | 9.981 GiB; 27.64% expert-byte hits |
| 24 GiB | 22.52% modeled read reduction | 62.66% | 17.981 GiB |

The results are simulation evidence only. No latency, throughput, physical
I/O, allocator, concurrent, or resident-dense runtime behavior was measured.

## Required gates for a future prototype

- Frozen F32 correctness and deterministic generated IDs remain unchanged.
- Dense and expert bytes are accounted under one explicit total-RAM budget.
- Peak working set and configured residency are reported separately and remain
  within the reviewed limits.
- The reference reader and strict global LRU remain the control path.
- Repeated full-runtime measurements show end-to-end improvement, not merely a
  logical-read or microbenchmark improvement.
- No quantization, mmap, prefetch, SIMD, GPU, or server work enters the task.
