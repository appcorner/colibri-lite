# ADR 0041: M5.3 expert-access prototype selection

- Status: accepted for next-task review
- Date: 2026-07-18
- Scope: M5.3-01 storage-path measurement

## Context

The Qwen3-30B-A3B F32 artifact stores one contiguous 18,874,368-byte payload
per expert, with contiguous gate/up/down projections and one shard per layer.
The production reader currently canonicalizes the tensor path, opens a shard,
seeks, allocates a fresh byte buffer, reads the payload, verifies SHA-256, and
closes the file for every expert miss. M5.2 runtime validation showed that the
larger cache reduces miss frequency for cacheable workloads, but misses remain
important for low-reuse workloads.

M5.3-01 added feature-gated counters and timing estimates, replayed the
authoritative miss traces, and ran a hash-checked isolated benchmark. It did
not modify the production reader or cache policy.

## Evidence

For a 64-request layer-47 miss subset, the current reader performed 64 opens,
64 seeks, 64 Rust read calls, 64 fresh allocations, and 1.208 GB of payload
copies. Measured time was 6.410 s, including 5.748 s of SHA-256 verification
and 0.543 s in the read interval. Persistent-handle/fresh-buffer access took
6.367 s in the same run; a reusable buffer took 5.996 s. These are warm-cache,
single-run directional measurements, not cold-cache or throughput claims.

Exact-adjacent range grouping reduced simulated operations by 3.5–6.5% with no
byte amplification. 4 KiB, 64 KiB, and 1 MiB thresholds added no grouping in
the selected matrix. One-layer batching introduced approximately 12.3–15.8x
byte amplification.

Full-shard mmap would require mapping approximately 115.96 GiB of virtual
address space across 48 files. A mapped lease would also require a new safe
ownership/lifetime contract and likely an audited Windows FFI boundary. The
current data does not show enough benefit to justify that complexity.

## Decision

Select **reusable aligned read buffers** as the one next storage-access
prototype. Keep it isolated and reversible. Compare it against the current
reader using identical payload ranges and SHA-256 verification, with separate
initialization and steady-state measurements. Do not change artifact layout,
cache policy, lease semantics, or numerical execution in the prototype task.

Do not select persistent handles, broad range coalescing, or mmap at this
stage. Persistent handles have low measured delta, broad coalescing over-reads
too much, and mmap has unresolved safe ownership and Windows-lifetime cost.

## Consequences

The next task can test allocation/copy overhead without introducing unsafe code
or a new dependency. The result may still be limited because the measured
reader path includes SHA-256 verification and this study did not isolate full
model matrix compute. A later mmap decision must be based on new controlled
measurements and a reviewed mapping ownership contract.
