# ADR 0036: Configurable expert-cache prototype

## Status

Accepted with limitations.

## Decision

Retain the existing safe Rust `ExpertCache` as the configurable memory
primitive. Callers supply an explicit payload-byte budget through
`ExpertStore::new`. The policy is strict deterministic global LRU; leases pin
entries; oversized entries are rejected; blocked eviction returns an error.

## Evidence

The ordered M5.1-00 trace was replayed through the production cache using the
M5.1-01 usable expert-cache budgets. Hits, misses, loads, evictions, bytes
avoided, and payload residency match the simulator at both 8 GiB and 16 GiB
operating points. Metadata/alignment differences are explicit and bounded.

## Limitations

The canonical full-model payload is not present in this checkout, so full-model
correctness, generated-token replay, startup, prefill, decode, working-set,
and latency evidence remain pending. The prototype is not declared production
ready and no physical I/O conclusion is drawn.

## Next candidate

Capture a broader representative trace corpus before selecting resident-dense,
mmap/coalesced access, compute profiling, or another cache policy experiment.
