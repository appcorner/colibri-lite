# ADR 0042: M5.3 reusable aligned read-buffer prototype

- Status: accepted for diagnostic use; not default
- Date: 2026-07-18
- Scope: M5.3-02 storage-path prototype

## Context

M5.3-01 measured one fresh payload allocation per expert miss and selected a
reusable aligned read buffer as the smallest reversible storage experiment.
The production reader owns the bytes returned to `ExpertStore` and verifies
each exact payload range with SHA-256. The artifact layout and strict global
LRU cache are frozen.

## Decision

Implement a feature-gated `ReusableAlignedBuffer` reader mode. Use one safe,
mutex-protected staging `Vec<u8>` for the synchronous load path. Grow but do
not shrink it, read and hash only the requested logical range, then copy the
verified bytes into an owned `Arc<[u8]>` before releasing the staging lease.
Keep `Reference` as the default and retain the existing public `Vec<u8>` reader
result contract. Do not introduce unsafe code, persistent handles, mmap,
coalescing, prefetch, or a new dependency.

The decoder’s safe byte-to-F32 conversion requires one-byte alignment, so the
alignment contract is explicitly `1`; no stronger alignment or raw pointer
cast is introduced. The current fixed expert payload is exactly 18,874,368
bytes, while the buffer lifecycle supports variable sizes and bounded growth
for the synchronous load.

## Evidence

Synthetic and byte-equivalence tests cover variable payload sizes, growth,
no-shrink reuse, repeated reads, truncation, hash failure, and file-handle
release. The isolated 64-miss benchmark changed 64 allocations to one and
reduced the measured read interval from 0.402 s to 0.277 s, but added the same
1,207,959,552-byte staging-to-owned copy volume.

The 24-run full-model matrix (six fixtures, 8/16 GiB, both reader modes) had
identical generated IDs, traces, cache counters, logical bytes, KV invariants,
and bounded residency. The reusable path was slower in 9/12 matched timing
comparisons and had a simple mean total time of 187.11 s versus 150.41 s for
the reference path. The filesystem cache was uncontrolled and samples are
directional, but the result does not justify making the reusable path default.

## Consequences

The prototype provides a safe diagnostic implementation and quantifies that
allocation elimination alone does not reduce end-to-end expert-load cost when
ownership still requires a full payload copy. The reference reader remains the
default. Persistent handles are not selected because M5.3-01 measured open and
seek time as small relative to hashing and payload processing. Compute remains
the next profiling target because scalar matrix-compute time was not isolated.
