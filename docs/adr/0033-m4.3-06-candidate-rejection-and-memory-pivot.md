# ADR 0033: Reject Full-Model Expert INT8 and Pivot to Memory Hierarchy

## Status

Accepted for M4.3-06 closure and roadmap planning.

## Decision

Reject symmetric INT8 per-output-channel expert weights as a full-model
production candidate. Retain the format for diagnostics and possible
selective-layer/selective-projection investigation only with new evidence.
Do not implement a quantized runtime.

Continue the project and make the first optimization experiment an F32
memory-hierarchy study using resident dense weights and trace-driven expert
cache sizing. The first step is simulation at 1/2/4/8/16/24/32 binary GiB;
runtime changes require separate review.

## Rationale

The candidate compresses experts by `4.200218x`, but its local weighted error
`0.1043549` becomes material propagated drift at Layer 1 and `152.1300659` at
Layer 47. The Thai fixture changes argmax, Top-20 overlap reaches `0.65`, and
Tier-A IDs `[1096, 374]` include ambiguous steps. Generation agreement alone
cannot override those quality risks. Per-tensor INT8 is worse; group-128 is
promising but lacks full-depth evidence.

The ik_llama Q4_K_M benchmark demonstrates useful CPU speed but is directional:
quantized storage, mmap, SIMD, threading, fused MoE, graph reuse, and optimized
kernels differ from colibri F32, and quality equivalence was not measured.

## Invariants

Canonical artifact identity/layout, ordered F32 arithmetic, RMSNorm and router
contracts, deterministic tie handling, frozen fixture IDs, selected
intermediate evidence, finite outputs, KV-cache allocation/no-overwrite, and
bounded memory remain mandatory for all future work.
